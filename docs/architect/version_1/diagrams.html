🚀 BRIDGE-API Flow Diagrams

Overview: These diagrams illustrate the complete banking-to-crypto bridge flows, including proper banking integration layers, compliance checks, and event-driven processing between traditional banking systems and blockchain networks.

💰 Balance Query Flow

Purpose: Shows how users can query their complete account balances, including both fiat (from banking system) and crypto (from blockchain) balances with proper banking integration.
%% Balance Query Flow: Bank/User check balances sequenceDiagram participant User as User/Client participant BridgeAPI as Bridge API participant DB as Database participant BankAPI as Banking API
(Open Banking/PSD2) participant Bank as Bank System participant Ethereum as Ethereum Network User->>BridgeAPI: GET /api/v1/balance?user_id=... Note over BridgeAPI, Bank: 1. Fetch Fiat Balance BridgeAPI->>BankAPI: GET /api/v1/banking/balance {user_id} BankAPI->>Bank: Query account balance Bank-->>BankAPI: Fiat balance BankAPI-->>BridgeAPI: Fiat balance data Note over BridgeAPI, Ethereum: 2. Fetch Crypto Balance alt custodial account BridgeAPI->>DB: Query crypto balances from ledgers DB-->>BridgeAPI: aggregated crypto balance else non-custodial BridgeAPI->>Ethereum: eth_call(balanceOf) Ethereum-->>BridgeAPI: token balance end Note over BridgeAPI, User: 3. Aggregate & Return BridgeAPI->>BridgeAPI: Aggregate fiat + crypto balances BridgeAPI-->>User: {fiat_balance, crypto_balance, total_balance}

📥 Deposit Flow

Purpose: Illustrates the complete deposit process from bank to blockchain, including banking integration, compliance checks, event-driven processing, and transaction settlement.
%% Deposit Flow: Bank -> Bridge -> Core Chain sequenceDiagram participant Bank as Bank System participant BankAPI as Banking API
(Open Banking/PSD2) participant BridgeAPI as Bridge API participant DB as Database participant Kafka as Kafka participant Worker as Bridge Worker participant Vault as Vault/HSM participant Ethereum as Ethereum Network Note over Bank, Ethereum: 1. Initiate Deposit Request Bank->>BankAPI: POST /api/v1/deposits {bank_ref, amount, account} BankAPI->>BankAPI: Validate bank account
Check KYC/AML BankAPI->>BridgeAPI: POST /api/v1/bridge/deposits {bank_ref, amount, user_id} Note over BridgeAPI, DB: 2. Bridge Processing BridgeAPI->>DB: Insert ledger (status=PENDING) BridgeAPI->>Kafka: Publish DepositEvent BridgeAPI-->>BankAPI: 202 ACCEPTED {bridge_ref} BankAPI-->>Bank: 202 ACCEPTED {bridge_ref} Note over Worker, Ethereum: 3. Crypto Processing Worker->>DB: SELECT ledger FOR UPDATE Worker->>Vault: Sign mint transaction Vault-->>Worker: Signed Tx Worker->>Ethereum: SendRawTransaction(signedTx) Ethereum-->>Worker: txHash Worker->>DB: Update ledger, insert tx Note over Worker, Bank: 4. Settlement & Notification Worker->>BankAPI: Callback {status=SETTLED, bridge_ref} BankAPI->>BankAPI: Update bank ledger BankAPI-->>Bank: Callback {status=SETTLED, bank_ref}

📤 Withdraw Flow

Purpose: Shows the withdrawal process from blockchain back to traditional banking, including banking integration, compliance checks, token burning, and bank payout coordination.
%% Withdraw Flow: User Redeem -> Bank Payout sequenceDiagram participant User as User/Client participant BridgeAPI as Bridge API participant DB as Database participant Kafka as Kafka participant Worker as Bridge Worker participant Vault as Vault/HSM participant Ethereum as Ethereum Network participant BankAPI as Banking API
(Open Banking/PSD2) participant Bank as Bank System Note over User, Bank: 1. Initiate Withdrawal Request User->>BridgeAPI: POST /api/v1/withdraw {amount, toBankAcc, user_id} BridgeAPI->>DB: Insert ledger (status=PENDING) BridgeAPI->>Kafka: Publish WithdrawEvent BridgeAPI-->>User: 202 ACCEPTED {bridge_ref} Note over Worker, Ethereum: 2. Crypto Burn Processing Worker->>DB: SELECT ledger FOR UPDATE Worker->>Vault: Sign burn transaction Vault-->>Worker: Signed Tx Worker->>Ethereum: SendRawTransaction(signedTx) Ethereum-->>Worker: txHash Worker->>DB: Update tx/ledger (status=PROCESSING) Note over Worker, Bank: 3. Bank Payout Processing Worker->>BankAPI: POST /api/v1/banking/payout {amount, account, bridge_ref} BankAPI->>BankAPI: Validate account
Check compliance BankAPI->>Bank: Initiate bank transfer Bank-->>BankAPI: Transfer confirmation BankAPI-->>Worker: Payout success Note over Worker, User: 4. Settlement & Notification Worker->>DB: Mark ledger SETTLED Worker->>User: Callback {status=SETTLED, bridge_ref}

Key Components:
  • Banking API: Open Banking/PSD2 integration with KYC/AML compliance
  • Bridge API: Main HTTP server handling REST API requests
  • Kafka: Event streaming for asynchronous processing
  • Worker: Background service processing blockchain transactions
  • Vault/HSM: Secure key management for transaction signing
  • PostgreSQL: Ledger and transaction storage
  • Ethereum: Blockchain network (Sepolia testnet)
